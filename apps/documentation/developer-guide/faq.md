# FAQ

Answers to the most common questions from contributors.

## Setup

### What are the system requirements?

- Node.js 20+
- pnpm (do not use npm or yarn)
- Docker (for the Postgres container)
- Git

Optional but recommended: [devenv](https://devenv.sh) + [direnv](https://direnv.net) for reproducible tooling via Nix.

### How do I start the project locally?

```bash
pnpm install
pnpm db:start    # starts Postgres container + runs migrations
pnpm dev         # starts API (port 5001) + frontend (port 3001)
```

See [Getting Started](../getting-started/README.md) for the full setup path.

### Where do environment variables come from?

The project uses [Phase](https://phase.dev) to inject secrets. For local development, copy `.env.example` to `.env` at the repo root and fill in the values. `packages/config` centralizes env parsing for both API and frontend — access values via `config.env.*`, never `process.env.*` directly.

### The API won't start. What should I check?

1. Is Docker running? (`docker ps`)
2. Is the database up? (`pnpm db:start`)
3. Does `.env` exist at the repo root?
4. Is port 5001 free? (`lsof -i :5001`)

### The frontend won't start. What should I check?

1. Is the API running? Frontend requires the API to fetch data.
2. Is port 3001 free? (`lsof -i :3001`)
3. Does `VITE_API_URL` in `.env` point to `http://localhost:5001`?

## Architecture

### Which layer should my code go in?

| Question | Answer |
|---|---|
| Is it a business rule or domain concept? | Domain layer |
| Is it orchestrating a business operation? | Application layer (use case) |
| Is it reading/writing a database or calling an external API? | Infrastructure layer |
| Is it handling an HTTP request? | Application layer (controller) |

### Why is there no DI container?

Dependencies are wired manually in each `*.module.ts` constructor. This is intentional: it keeps the DI graph explicit, readable, and greppable without any reflection magic or framework lock-in. The trade-off is verbosity in module files.

### Why does each module have its own `DecoratorRouter`?

Each module instantiates `DecoratorRouter` locally so that route metadata from one controller never bleeds into another. The router reads decorator metadata off the controller class and produces an isolated Express `Router`. See [Decorator System](architecture/decorator-system.md) for details.

### Where are domain entities defined?

In `src/modules/<module>/domain/entities/`. Entities are plain TypeScript classes with no framework imports. They may contain business methods (e.g. `user.isEmailVerified()`), but they never talk to the database or Express.

### What is the `tmdb_fetch_status` table?

It tracks whether a given TMDB record has been fetched and stored locally. When a content request comes in, the composite repository checks this table: if the content is already cached in Postgres, it returns it directly; otherwise, it fetches from TMDB, writes to Postgres, and marks the status. See [TMDB Integration](architecture/tmdb-integration.md).

## Development

### How do I add a new route to an existing module?

See [Adding an Endpoint](guides/adding-an-endpoint.md) — the fastest path is 5 steps.

### How do I create a completely new feature module?

See [Creating a Module](guides/creating-a-module.md) — 13 steps from DB schema to registered module.

### Do I need to regenerate the SDK after changing an endpoint?

Yes. Run:

```bash
pnpm generate-sdk
```

The SDK in `packages/api-sdk/src/generated/` is auto-generated by Orval from `apps/api/api-documentation.json`. Never edit generated files manually.

### Can I use `console.log` in the API?

`console.log` is permitted but `console.error`, `console.warn`, and `console.info` are flagged by Biome in non-logger packages. Use `@packages/logger` instead:

```typescript
import { logger } from "@packages/logger";
logger.info("User logged in", { userId });
```

### My Biome lint check fails. How do I fix it?

```bash
pnpm format    # auto-fixes formatting and safe lint errors
pnpm lint      # shows remaining errors that need manual fixes
```

The most common causes: missing `import type`, `let` instead of `const`, unused variables, `==` instead of `===`.

## Testing

### What test framework is used?

**Vitest**. There is no Jest, no `@testing-library`, and no DOM environment. Tests run in Node.

### How do I run a single test file?

```bash
cd apps/api && pnpm vitest run src/modules/auth/application/use-cases/login.usecase.test.ts
```

### How do I mock a repository?

Use the factory function from `domain/interfaces/*.repository.mock..ts`:

```typescript
const repo = createMockedUserRepository();
// override a single method:
const repo = createMockedUserRepository({
  findById: () => Promise.resolve(null),
});
```

See [Writing Tests](guides/writing-tests.md) for the full guide.

### Tests pass locally but the pre-commit hook fails. Why?

The hook runs `pnpm lint`, `pnpm check-types`, `pnpm test`, and `pnpm build` in sequence. A test that passes in watch mode may still break the build step. Run the full suite manually:

```bash
pnpm lint && pnpm check-types && pnpm test && pnpm build
```

## Authentication

### How does the dual-token auth flow work?

- **Access token**: 15-minute JWT, sent as `Authorization: Bearer <token>` on every request, stored in frontend memory (Zustand + `sessionStorage`)
- **Refresh token**: 7-day JWT, stored in an `httpOnly` cookie, used only for `/auth/refresh`

On login, both tokens are issued. Every successful `/auth/refresh` rotates both tokens. On logout, the refresh token is revoked server-side in the `refresh_tokens` table.

### How do I protect an endpoint?

Add `@Protected()` to the controller method:

```typescript
@Get("/me")
@Protected()
getMe = asyncHandler(async (req: Request, res: Response) => {
  const userId = req.user!.id;
  // …
});
```

`DecoratorRouter` will apply `authMiddleware` to that route automatically.

## Database

### How do I create a migration?

```bash
# 1. Edit src/database/schema.ts
cd apps/api && pnpm db:generate    # generates migration file
cd apps/api && pnpm db:migrate     # applies it
```

Review the generated file in `src/database/migrations/` before applying.

### Can I roll back a migration?

Drizzle does not support automatic rollbacks. Write a new migration that reverses the change, or restore from a backup.

### How do I inspect the database visually?

```bash
cd apps/api && pnpm db:studio    # opens Drizzle Studio in your browser
```

## Contributing

### What commit format should I use?

```
<type>(<scope>): <short description>
```

Example: `feat(auth): add email verification`. See [Commit Conventions](guidelines/commit-conventions.md).

### Can I skip the pre-commit hook?

Only in a genuine emergency (e.g. a docs-only commit when the DB is not running). Use `--no-verify` sparingly and only when you understand what you are bypassing.

### I found a bug. How do I report it?

Open a GitHub issue using the **Bug Report** template. Include steps to reproduce, expected vs actual behavior, and your environment (OS, Node version).

### Where do I ask questions that aren't in this FAQ?

[GitHub Discussions](https://github.com/mathieusouflis/HETIC-projet-cinema/discussions) — open a new discussion in the Q&A category.
